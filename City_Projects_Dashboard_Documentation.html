City Projects Dashboard Documentation

This document explains the structure and meaning of the data used in the City Projects Dashboard. This dashboard provides insights into various city projects, including their status, budget, feedback, and other key metrics, to help track progress and resources effectively.

1. Project Overview

The "Project Overview" section provides key information about each project in the city, including:

  • Project Name: The name or identifier for each city project (e.g., Project A, Project B).
  • Status: The current state of the project, categorized as:
    • In Progress: The project is currently being worked on.
    • Completed: The project has been finished.
    • Not Started: The project is planned but has not yet begun.
  • Budget: The allocated budget for each project in USD.
  • Completion Date: The anticipated or actual date by which the project is expected to be completed.

This information helps users see the overall scope, progress, and financial allocation for all city projects.

2. Feedback Statistics

The "Feedback Statistics" section captures how feedback is processed for projects. It includes:

  • Unseen Feedback: The number of feedback entries not yet reviewed.
  • Seen Feedback: Feedback entries that have been reviewed but may not have been acted upon yet.
  • Forwarded Feedback: Feedback that has been sent to the relevant team or department for further action.
  • Resolved Feedback: Feedback items that have been addressed and closed.

This provides an understanding of how much feedback has been received, reviewed, and resolved for city projects.

3. Project Statistics

This section summarizes high-level statistics for all projects, including:

  • Total Projects: The total number of city projects tracked on the dashboard.
  • Ongoing Projects: Projects that are currently in progress.
  • Completed Projects: Projects that have been finished.
  • Delayed Projects: Projects that may be past their estimated completion dates (data not specified here).
  • Total Expenditure: The sum of the budget for all projects, displayed in crores (Cr) for large budgets.
  • Average Delay: This placeholder metric indicates how far behind projects are on average, if any delay data is available.

These statistics provide a quick overview of the project's progress and financial usage.

4. Charts Data

Various visualizations are generated to help interpret project data. These include:

  • Bar Chart: Displays each project's budget to compare financial allocation across projects.
  • Doughnut Chart: Shows the distribution of projects by status (Completed, In Progress, Not Started) for a quick view of project stages.
  • Line Chart: Typically used to visualize feedback or resolution rates over time.
  • Pie Chart: Represents resource or departmental allocation for project activities, such as Marketing, Development, HR, and Sales.
  • Horizontal Bar Chart: Shows task breakdowns and completion for selected tasks within projects.

These charts enable users to analyze project progress, budget allocation, task status, and feedback over time visually.

By understanding these data points and visualizations, users can better track project performance, budgets, and feedback management across the city.
